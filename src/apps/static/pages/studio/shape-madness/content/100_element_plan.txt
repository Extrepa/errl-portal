# Master plan for 100 psychedelic HTML/CSS/JS math toys

You want a living lab: a mobile-friendly gallery that batches generation of insane math toys with preview, live code, description, and tweakables. Below is the master plan, the generation pipeline, the shared template, and 100 idea specs with concise parameters so we can spin them up fast.

---

## Master plan overview

- **Goal:** Build a modular gallery where each toy renders a preview, shows live code, explains the math, and exposes tweak knobs—ready for batch generation, export, and remix.
- **Structure:** One master HTML file orchestrates tabs, lazy init, and a shared template. Each toy is a module with:
  - Preview canvas/SVG/div
  - Minimal code block synced to the produced output
  - Short description
  - Tweak controls (sliders, color pickers, toggles)
- **Batching:** Generate in sets of 10 (10 panels per route), then paginate; each panel initializes when opened to avoid background load.
- **Performance:** Prefer Canvas and CSS gradients; use SVG for lattice/tiling; throttle animation on mobile; detect prefers-reduced-motion; cap pixel density.

---

## Generation pipeline

1. **Registry:** Define a registry array of 100 toy specs (id, title, mode, description, defaults, init function name).
2. **Template:** Use a single panel template with slots: Title, Preview Stage, Tools, Live Code, Description.
3. **Lazy init:** On tab selection, inject preview node, controls, and wire the init function; never render off-screen.
4. **Code sync:** Render the live code block from the actual init function text or a generated snippet that matches the current params.
5. **Tweak handling:** Each control updates a params object and triggers a redraw; maintain a small “preset” dropdown for fast variations.
6. **Batch routes:** Route segments (#page=1…10) for 10 toys per view; preload only the current and adjacent toys.
7. **Export hooks:** For Canvas—PNG export; for SVG—inline SVG export; for CSS—copy styles. Include a “Copy code” button.
8. **Mobile UX:** Sticky tab bar, scroll-snap panels, big sliders, tap gestures, low-latency pointer events.
9. **Safety:** Clamp loop iterations, resolution, and particle counts; expose “Quality” slider that scales work without altering look.

---

## Template schema

- **Panel header**
  - **Title:** Name and id
  - **Tools:** Controls bound to params
- **Preview stage**
  - Canvas/SVG/Div with class “stage”
- **Live code**
  - A synchronized code block showing the generating code (copyable)
- **Description**
  - A brief “what’s happening” and “what you can tweak”
- **Preset bar**
  - Quick buttons to switch modes

> Tip: We can auto-generate most panels from declarative specs, then only hand-write the math core per toy.
