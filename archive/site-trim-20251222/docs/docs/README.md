# Project Docs

Purpose: keep planning, notes, and decisions separate from the app code. **No builds or source code files in here.**

## Directory Structure

### Core Documentation
- **journal/** — Dated work logs, `TEAM_UPDATE_*` files, and migrated `notes/`
  - Format: `YYYY-MM-DD.md` or `YYYY-MM-DD-topic.md`
  - Contains: Daily logs, team updates, checkpoint summaries
- **team/** — Onboarding, workflow, and safety guides
  - Developer guides, AI prompt templates, motion graphics guide
- **adr/** — Architecture Decision Records
  - Format: `0001-*.md`, one decision per file
  - Template: Context → Decision → Consequences → Alternatives
- **architecture/** — Diagrams, schemas, and design docs
- **assets/** — Images used by docs (not shipped with the app)

### Project Planning
- **roadmap.md** — Near-term plan and milestones
- **changelog.md** — Human-friendly notable changes (dev-facing)
- **catalog/** — Component inventory and planning documents
  - Component rips catalog, inventory tracking, normalization plans

### Specialized Documentation
- **dev/** — Development notes and plans
- **perf/** — Performance benchmarks and analysis
- **reorg/** — Reorganization plans and structure reviews

## Conventions

- Use **YYYY-MM-DD** filenames for journal entries
- ADR template: Context → Decision → Consequences → Alternatives
- Link to source files with relative paths (e.g., `../src/fx/errl-bg.ts`)
- Keep production assets in `src/` (no builds in `docs/`)
- All source code files belong in `src/`, not `docs/`

## Directory Purpose Summary

### `docs/` vs `src/` vs `archive/` vs `dist/`

- **`docs/`** — Documentation ONLY. No source code, builds, or HTML app files
- **`src/`** — All source code (TypeScript, React, HTML, CSS, JS). This is what gets built
- **`archive/`** — Archived content including legacy standalone pages (not part of main build)
- **`dist/`** — Build output directory (generated by Vite from `src/`)

## Related Root Docs

- `README.md` — Repo overview and dev commands (still at repo root)
- `docs/STATUS.md` — Consolidated project status and next steps (if exists)
