## How to Use This Template

When you return and find issues, create a new file or add to this one with:

1. **Issue Description**: What's wrong?
2. **Steps to Reproduce**: How to see the issue
3. **Expected Behavior**: What should happen
4. **Actual Behavior**: What actually happens
5. **Browser/Device**: Where it occurs
6. **Priority**: Critical / High / Medium / Low
7. **Status**: Open / In Progress / Fixed / Won't Fix
